Die Ollama CORS-Konfiguration hängt von Ihrem Betriebssystem und der Art ab, wie Sie den lokalen Server ausführen.
Auf dieser Seite finden Sie alle Informationen, die mit Hilfe der ThunderAI-Benutzer gesammelt wurden.
Wenn Sie eine neue Konfiguration beitragen möchten, reichen Sie bitte ein neues Issue ein.

Wenn Sie CORS auf dem Server nicht konfigurieren können oder keinen Zugriff darauf haben, können Sie ThunderAI jetzt die Berechtigung "Alle URLs" erteilen, um CORS-Probleme zu vermeiden.
Verwenden Sie die entsprechende Schaltfläche auf der Optionsseite, wie unten gezeigt.

Docker Desktop

Wenn Sie einen neuen Container starten, öffnen Sie die "Optionalen Einstellungen" und fügen Sie eine "Umgebungsvariable" mit den folgenden Werten hinzu:
  • Variable: OLLAMA_ORIGINS
  • Wert: moz-extension://*

Docker-Befehlszeile

Wenn Sie einen neuen Container starten, fügen Sie diese Option dem Docker-Befehl hinzu: 
-e 'OLLAMA_ORIGINS=moz-extension://*'

Docker Compose

Dank an Amras2322

Wenn Sie den Ollama-Dienst konfigurieren, fügen Sie die Option OLLAMA_ORIGINS im Abschnitt environment der Datei docker-compose.yml hinzu: 
services:
  ollama:
    image: ollama/ollama:latest
    container_name: ollama
        [...]
    environment:
    - OLLAMA_ORIGINS=moz-extension://*

Ollama unter Windows

Dank an actXc

  1. Windows-Einstellungen öffnen (Windows-Taste + I)
  2. Nach "Umgebungsvariablen" suchen
  3. Auf "Systemumgebungsvariablen bearbeiten" klicken
  4. Im neuen Fenster auf "Umgebungsvariablen" klicken
  5. Unter "Systemvariablen" auf "Neu" klicken
  6. Variablenname: OLLAMA_ORIGINS
  7. Variablenwert: moz-extension://*
  8. Mit "OK" bestätigen
  9. PC neu starten

Ollama unter macOS

Dank an jatoko

  1. Terminal öffnen
  2. Folgenden Befehl im Terminal ausführen: 
    launchctl setenv OLLAMA_ORIGINS "moz-extension://*"
  3. Ollama (neu)starten (z.B. beenden und neu starten)
