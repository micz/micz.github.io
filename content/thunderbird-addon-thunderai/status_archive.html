This is the archive page for known issues in interacting with the ChatGPT web interface.
Go to the actual status page.


2024/09/11
A change in the ChatGPT web interface prevents again the addon from working.
See #133.

From version 2.1.4 the issue has been fixed, update the addon directly from Thunderbird.


2024/09/04
A change in the ChatGPT web interface prevents the addon from working.

From version 2.1.2 the issue has been fixed, update the addon directly from Thunderbird.


2024/08/21
A change in the ChatGPT web interface prevents the addon from sending automatically the prompt.

Version 2.0.7 fixes the issue.
Meanwhile just press the button to send the prompt.


2024/08/14
A change in the ChatGPT web interface prevents the addon from correctly importing the text when replying or substituting.

Version 2.0.2 fixes the issue.

Starting from version 2.0.2, to use a response in a reply, the user must select the text.
A single click should suffice, but it is also possible to manually select the desired text.

See #103.


2024/07/19
A change in the ChatGPT web interface prevents the addon from correctly importing the text when replying or substituting, unless the "Keep formatting" option is used.

Version 1.2.1 fixes the issue. See #87.


2024/05/15
A change in the ChatGPT web interface prevents to send the prompt. You have to click the send button.

The imported text has a ChatGPT string in front of it that you have to remove manually.
From version 1.0.8 onwards, the issues are fixed (#57 and #60).


2024/05/09
A change in the ChatGPT web interface prevents the button to use the answer automatically from appearing.
Version 1.0.6 fixes the issue (#43).


2024/05/02
If using ChatGPT-4 model, maintain the ChatGPT window width to 700.
With a larger width, it may not function properly.
See issue #38.
