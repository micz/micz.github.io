La configuration CORS d'Ollama dépend de votre système d'exploitation et de la façon dont vous exécutez le serveur local.
Sur cette page, vous trouverez toutes les informations recueillies avec l'aide des utilisateurs de ThunderAI.
Si vous souhaitez contribuer avec une nouvelle configuration, veuillez soumettre un nouveau ticket.

Si vous ne pouvez pas configurer CORS sur le serveur ou n'y avez pas accès, vous pouvez désormais accorder la permission "Toutes les URLs" à ThunderAI pour éviter tout problème CORS.
Utilisez le bouton correspondant sur la page des options, comme indiqué ci-dessous.

Docker Desktop

Lors du démarrage d'un nouveau conteneur, ouvrez les "Paramètres optionnels" et ajoutez une "Variable d'environnement" avec les valeurs suivantes :
  • Variable : OLLAMA_ORIGINS
  • Valeur : moz-extension://*

Docker en ligne de commande

Lors du démarrage d'un nouveau conteneur, ajoutez cette option à la commande docker : 
-e 'OLLAMA_ORIGINS=moz-extension://*'

Docker Compose

Merci à Amras2322

Lors de la configuration du service Ollama, ajoutez l'option OLLAMA_ORIGINS dans la section environment du fichier docker-compose.yml : 
services:
  ollama:
    image: ollama/ollama:latest
    container_name: ollama
        [...]
    environment:
    - OLLAMA_ORIGINS=moz-extension://*

Ollama sous Windows

Merci à actXc

  1. Ouvrir les Paramètres Windows (touche Windows + I)
  2. Rechercher "variables d'environnement"
  3. Cliquer sur "Modifier les variables d'environnement système"
  4. Dans la nouvelle fenêtre, cliquer sur "Variables d'environnement"
  5. Sous "Variables système", cliquer sur "Nouveau"
  6. Nom de la variable : OLLAMA_ORIGINS
  7. Valeur de la variable : moz-extension://*
  8. Confirmer avec "OK"
  9. Redémarrer le PC

Ollama sous Linux

Merci à lakonis

### Édition de /etc/systemd/system/ollama.service.d/override.conf
### Tout ce qui se trouve entre ici et le commentaire ci-dessous deviendra le contenu du fichier drop-in

[Service]
Environment="OLLAMA_ORIGINS=moz-extension://*"

### Les modifications en dessous de ce commentaire seront ignorées
[...]
Testé sous Linux Manjaro

Ollama sous macOS

Merci à jatoko

  1. Ouvrir le terminal
  2. Exécuter la commande suivante dans le terminal : 
    launchctl setenv OLLAMA_ORIGINS "moz-extension://*"
  3. (Re)démarrer Ollama (par ex. quitter et relancer)
